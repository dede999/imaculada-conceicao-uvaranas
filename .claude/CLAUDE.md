# Project instructions — Claude Code

Read `PROJETO.md` at the root before any task. It contains architecture, design, and stack decisions that must not be reverted without explicit discussion.

---

## Stack — non-negotiable rules

- **Framework**: Nuxt 3 with `@nuxt/content`
- **Styling**: Tailwind CSS — fixed light mode only. Never install `@nuxtjs/color-mode`. Never use `prefers-color-scheme`.
- **i18n**: `@nuxtjs/i18n` — never hardcode strings in templates. Every UI string goes in `locales/pt-BR.json`.
- **TypeScript**: required in composables and utilities. Vue components may use `<script setup lang="ts">`.

## Colors — fixed tokens (no theme variables)

Amber palette (Franciscan — habit, ochre, sand):
- `--fr-950: #412402` / `--fr-800: #633806` / `--fr-600: #854F0B`
- `--fr-400: #BA7517` / `--fr-200: #EF9F27` / `--fr-50: #FAEEDA`

Teal palette (convent — cloister green, moss):
- `--cv-950: #04342C` / `--cv-600: #0F6E56` / `--cv-400: #1D9E75`
- `--cv-50: #E1F5EE`

Defined in `assets/css/tokens.css` and imported globally via `nuxt.config.ts`.

## Code conventions

- Composables in `composables/` with `use` prefix
- Business logic never in components — always in composables
- Page components in `pages/`, reusable components in `components/`
- Content in `content/` following the frontmatter schema defined in `PROJETO.md`
- Never manually edit files in `content/transparencia/` — generated by CI script

## Tau SVG

The Franciscan Tau is in `assets/tau.svg`. Always use with `fill="currentColor"` and control color via CSS on the parent element. Never recreate the path inline in components.

## Critical components

### `StatusPanel.vue`
Uses `setInterval` + `useNow` (VueUse) to calculate live mass and confession status. Data comes from chapel frontmatter via Nuxt Content. Do not fetch external data.

### `CatechismPill.vue`
- Desktop: tooltip on hover
- Mobile: bottom sheet with `Teleport` to `body`
- Detect breakpoint via `useBreakpoints` (VueUse)
- Bottom sheet closes on outside tap OR after 4 seconds (`setTimeout`)

### `InstagramFeed.vue`
- Desktop: 3-column grid
- Mobile: pure CSS carousel with `scroll-snap-type: x mandatory`. No external library.
- Provider: Behold.so

## Event display logic (home)

```typescript
const showEvent = computed(() =>
  events.value.some(e =>
    e.status === 'active' && new Date(e.date) > new Date()
  )
)
```

If `showEvent` is false, the event card is hidden and the news + announcement cards expand to fill the space.

## Financial script (Go or Rust)

Located in `scripts/parse_sheet/`. Always compile the binary before running — never use `go run` or `cargo run` in CI. Output goes to `content/transparencia/dados.json`.

## Supabase (tithe module)

Schema and RLS policy documentation in `docs/dizimo.md`. All logic centralized in `composables/useDizimo.ts`. Never spread Supabase calls across components.

## What not to do

- Do not install carousel libraries — CSS scroll-snap is enough
- Do not install tooltip libraries — implement with CSS + VueUse
- Do not use browser `alert()`, `confirm()` or `prompt()`
- Do not hardcode UI text in templates
- Do not create color variables outside `tokens.css`
- Do not manually commit to `content/transparencia/`
